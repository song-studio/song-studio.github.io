# 世界杯球迷卡牌赛 运营回归手册

## 目标
每次改版后，不再靠人工“边点边找 bug”，而是先做固定回归，再上线。

## 上线前固定动作
1. 进入项目目录：
   - `cd /Users/song/Desktop/Codex/song-studio.github.io/song-studio.github.io-work/2026-FIFA-World-Cup`
2. 执行一键自检：
   - `./qa/smoke-check.sh`
3. 只有在看到 `PASS` 后才允许推送。

## 本脚本覆盖的关键风险
- 中转页手机端横向晃动（viewport / overflow）
- 彩蛋卡误设主队
- 每日开卡上限失效
- 手机端八张稀有叙事卡“显示已解锁但无法提交”
- 比赛日没数据时没有预测入口（演示入口兜底）
- `matches.json` 结构损坏

## 赛中每日维护（无 API 手动模式）
1. 打开 `data/matches.json`
2. 更新 `today` 为当天比赛（含 `id/home/away/homeId/awayId/time`）
3. 赛后补 `winner`（`home` / `draw` / `away`）
4. 保存后刷新页面，系统自动结算星级与奖励

## 异常处理
- 如果自检失败：禁止上线，先修失败项
- 如果线上异常：先回滚到上一个 commit，再排查

## 版本原则
- 只做小步改动（一次只改一类功能）
- 每次改动都要重新跑 `./qa/smoke-check.sh`
