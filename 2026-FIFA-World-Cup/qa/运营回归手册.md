# 世界杯球迷卡牌赛 运营回归手册

## 目标
每次改版后，不再靠人工“边点边找 bug”，而是先做固定回归，再上线。

## 上线前固定动作
1. 进入项目目录：
   - `cd /Users/song/Desktop/Codex/song-studio.github.io/song-studio.github.io-work/2026-FIFA-World-Cup`
2. 执行一键自检：
   - `./qa/smoke-check.sh`
3. 只有在看到 `PASS` 后才允许推送。

## 什么算“发布前”
- 任何文件改动都算：HTML/CSS/JS/JSON/文案/按钮文案/交互逻辑。
- 任何准备 `git push` 的动作都算一次发布前检查。
- 赛中即使只改一场比赛数据，也建议执行一次自检。

## 本脚本覆盖的关键风险
- 中转页手机端横向晃动（viewport / overflow）
- 彩蛋卡误设主队
- 每日开卡上限失效
- 手机端八张稀有叙事卡“显示已解锁但无法提交”
- 比赛日没数据时没有预测入口（演示入口兜底）
- `matches.json` 结构损坏

## 赛中每日维护（无 API 手动模式）
1. 打开 `data/matches.json`
2. 更新 `today` 为当天比赛（含 `id/home/away/homeId/awayId/time`）
3. 赛后补 `winner`（`home` / `draw` / `away`）
4. 保存后刷新页面，系统自动结算星级与奖励

## 赛中 10 项极简人工点击清单（桌面+手机）
1. 首页可打开，Hero 区按钮正常跳转（赛程页 / 卡牌页）。
2. 中转页可打开，无左右晃动，两个入口按钮可点。
3. 主队三选一可正常选卡，且彩蛋卡不能设为主队。
4. 每日手气可正常执行“三翻二选”，领取后状态正确更新。
5. 彩蛋卡逻辑正常：未点亮时提示正确；点亮后可触发一次剧场并加能量。
6. 留言板可提交，首次留言奖励只发一次，重复提交不重复奖励。
7. 八张稀有叙事题：已解锁题可提交并锁定；刷新后答案仍在。
8. 比赛日预测入口正常：无 today 时有“加载预测演示”；有 today 时显示比赛列表。
9. 单场预测可提交并写入状态；赛果回填后可自动结算（命中/平局/失败规则正确）。
10. 中英文切换正常，无关键按钮失效、无明显中英混杂文案。

```bash

```
然后按“赛中 10 项极简人工点击清单”逐项执行并回报：`PASS/FAIL + 截图/描述`。

## 异常处理
- 如果自检失败：禁止上线，先修失败项
- 如果线上异常：先回滚到上一个 commit，再排查

## 版本原则
- 只做小步改动（一次只改一类功能）
- 每次改动都要重新跑 `./qa/smoke-check.sh`
