# 世界杯比分与赛事中心修复设计

## 目标

恢复截至 2026 年 6 月 27 日 11:00（北京时间）的世界杯比分、积分榜和赛事入口，同时保留首页 Hero、MCP、留言板及球迷卡牌赛。

## 信息架构

首页继续承担赛事总入口：Hero 下方先显示“赛事中心”，提供今日比赛、完整赛程、小组积分榜和淘汰赛四个入口；12 个小组、更多功能、留言板保持原位。小组详情继续显示六场赛程，并直接在队名之间显示比分。卡牌赛继续只消费 `matches.json` 的 `today` 数组，不改奖励卡、纸雕框、勋章和下载逻辑。

## 数据流

`data/matches.json` 成为赛事数据源：`matches` 保存 72 场完整小组赛，`today` 只保存北京时间当天场次。`data/standings.json` 由已完场比赛生成 12 组积分。页面按比赛 ID 和球队 ID 匹配数据；中文队名只用于显示，球队 ID 以 `data/cards.json` 为准。

## 容错

单场数据缺失或球队 ID 无效时，页面保留该场赛程并显示“待赛”，不会中断其他场次渲染。比分加载失败时，赛事中心显示明确的错误状态；小组赛静态赛程仍可浏览。

## 排名规则

积分按胜 3、平 1、负 0 计算，组内依次按积分、净胜球、进球数排序。FIFA 同分后的相互战绩、公平竞赛积分和抽签规则暂不自动计算；若前三项仍相同，保持球队在小组中的原始顺序，并在数据说明中写明范围。

## 验证

自动校验 HTML 内联脚本语法、72 场完整性、球队 ID、比分与胜负一致性、积分榜计算结果。浏览器验证首页、小组详情、今日比赛、积分榜、淘汰赛入口、移动端布局和卡牌赛控制台。

## 设计依据与决策记录

当前网站是主参考：沿用深蓝玻璃卡片、金色赛事强调色、现有字号和圆角，不引入新的视觉语言。用户交接要求决定赛事中心的四个入口；`cards.json` 决定球队身份；现有 `fan-cards.html` 决定 `today` 数组必须保持“当天比赛”的语义。

| 决策 | 依据 | 原因 |
| --- | --- | --- |
| Hero 后先放赛事中心 | 用户要求、现有首页层级 | 比分成为进行中赛事的首要任务 |
| 复用现有玻璃卡片与金色 | 当前生产页面 | 降低改动风险并保持品牌一致 |
| 完整赛程与今日比赛分字段 | 卡牌赛现有数据流 | 避免卡牌赛一次渲染全部 72 场 |
| 以 ID 匹配比分 | 既有覆盖错误与交接要求 | 队名翻译或别名变化不会破坏渲染 |
