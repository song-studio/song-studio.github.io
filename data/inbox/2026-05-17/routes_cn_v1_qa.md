## 五、数据质量问题（需关注）

### 1. 海拔数据精度（PARTIAL）
部分线路的 elevation_gain/loss 是基于整体路程估算的爬升累积值，并非精确的 GPS 实测数据。对于超长线路（如新藏线、川藏南线），爬升数据是在已知垭口高度基础上推算的，误差可能在 5-10% 范围内。

### 2. 典型时长数据（PARTIAL）
typical_duration_h 值参考了社区商业队行程时长，个体差异可能很大（尤其对 solo 方式或逆穿）。所有时长标注的是一个"标准商业团/常规节奏"的参考值。

### 3. GPX 链接缺失（PARTIAL）
120 条线路中仅 48 条有直接 GPX 链接。原因是两步路等平台的轨迹链接是动态/登录后可查的，无法提供稳定的直链。但每条线路均在两步路（2bulu.com）或行者等平台上有可搜索到的轨迹记录。

### 4. 部分数据来源时间较旧（PARTIAL）
部分来源文章写于 2018-2022 年之间，线路本身未变，但路况（如墨脱公路已通、丙察察已铺装）已发生实质改变。"现状数据"（如季节窗口、信号覆盖）在 last_verified_date 字段标注了最近验证时间，可能与来源文章时间不同。

### 5. 登山 permit_required 不统一（PARTIAL）
商业登山队报名本身包办审批，部分山峰自主攀登也需要向中国登协申请。permit_required=true 的判定标准为"法律/政策层面明确要求审批"，但具体操作层面，通过商业队报名时由带队方处理。

### 6. 摩旅路线 elevation_gain_m 估算偏大（PARTIAL）
摩托车走公路，爬升只走铺装路面的垭口翻越，实际累积爬升应低于同路线的骑行版本。但摩旅数据为了保持线路标识一致性，参考了同线路的骑行数据，实际爬升可能偏大 10-20%。

## 八、缺失数据说明

**gpx_url = null 的原因：**
- 两步路/行者的轨迹链接需要登录才能稳定访问，不提供直接外链
- 部分线路（如徽杭古道、漓江徒步）不是常规的"重装穿越"，休闲徒步者一般不生成 GPX
- 登山线路的 GPX 多为商业队内部文件，不公开发布

**region.city_prefecture/county = null 的原因：**
- 跨省/跨多个地州市的超长线路（如川藏线、京杭大运河）
- 线路终点跨行政区域边界（如南太行走廊涉及河南/山西）

## 九、建议后续优化方向

1. **补充 GPX 链接**：接入两步路 API 或行者 API 自动获取轨迹链接
2. **加入 P/I (Page/Implement) 难度评级**：当前仅在 risk_tags 中隐含难度信息，可另设 `difficulty_rating` 字段
3. **加入经济数据**：添加参考预算、门票、商业队费用等字段
4. **图片/封面**：添加线路缩略图 URL 用于路线决策台展示
5. **实时路况**：对接交通/气象 API 提供当前路况信息
