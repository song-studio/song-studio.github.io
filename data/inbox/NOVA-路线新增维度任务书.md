# Nova 路线数据 — 新增维度任务书

> 下发时间：2026-05-24
> 这是对 `NOVA-路线接管任务书.md` 的补充——在完成基础校对后，新增以下数据维度。

---

## 新增一：许可证/政策信息（国内关键）

中国户外路线的政策风险是决策第一关——路线被封了，其他数据都没用。

每条国内线路标注：
| 字段 | 内容 |
|------|------|
| access_status | 开放 / 限制（需报备） / 封闭（官方禁止穿越） / 季节性开放 |
| permit_required | 是/否 |
| permit_how | 获取方式（如"需在景区管理处登记""需报备当地林草局"） |
| regulation_note | 相关政策说明（如"鳌太全线禁穿，违者罚款5000"） |
| last_verified_date | 政策确认日期 |

已知封闭/限制线路（优先标注）：
- 鳌太穿越（全线禁穿）
- 夏塔古道（限制进入，需报备）
- 珠峰东坡（需报备+配向导）
- 他念他翁（部分段军事管制）
- 独库公路（季节性封路）
- 新藏线（边境证+部分段军事管制）

---

## 新增二：交通到达方式

"怎么去"是户外决策的第一问。每条线路标注：

| 字段 | 示例 |
|------|------|
| trailhead_access | "丽江客运站→虎跳峡镇，班车1.5h，¥35" / "仅包车/自驾" / "无公共交通，需联系当地司机" |
| return_from_endpoint | 终点如何返回（班车/包车/原路返回/环形无需返回） |
| nearest_city | 最近城市名+距离 |
| nearest_transport_hub | 最近机场/火车站+距离 |

---

## 新增三：海拔剖面 + 关键节点

目前缺这个，户外用户最需要。每条线路：

| 字段 | 说明 |
|------|------|
| elevation_profile | `[[距离km, 海拔m], ...]` 最少 10 个采样点 |
| key_altitude_points | 起点海拔 / 最高点海拔+位置 / 终点海拔 / 主要垭口海拔 |
| cumulative_elevation | 累计爬升（区别于 elevation_gain_m 的净爬升） |

有 GPX 的用 GPX 解析，没有的用 Google Earth / 公开攻略估算，标注来源。

---

## 新增四：营地/住宿分布

长途线路的营地规划是关键决策：

| 字段 | 说明 |
|------|------|
| campsites | 营地列表：名称、大致里程位置、海拔、水源、容量（如"10-15顶帐篷"） |
| huts | 木屋/客栈列表（如有） |
| camping_restriction | "任意扎营" / "仅指定营地" / "禁止扎营（自然保护区）" |

---

## 新增五：信号覆盖地图

标注得越具体越好：

| 字段 | 说明 |
|------|------|
| signal_coverage | 逐段标注 移动/电信/联通 信号情况 |
| signal_dead_zones | 无信号区段（里程范围） |
| last_signal_point | 进入无信号区前的最后一个有信号点 |

格式示例：移动 全程有信号 / 电信 大本营后无信号 / 联通 大部分无信号

---

## 新增六：线路变体

同一条线路常有不同走法：

| 字段 | 说明 |
|------|------|
| route_variants | 变体列表（如"2日速穿版 / 3日标准版 / 5日摄影版"） |
| variant_differences | 各变体的里程/爬升/营地差异 |

---

## 执行优先级

| 优先级 | 维度 | 原因 |
|--------|------|------|
| P0 | 许可证/政策 | 安全第一，封闭线路必须标 |
| P0 | 交通到达 | 用户第一问 |
| P1 | 营地/住宿 | 长途刚需 |
| P1 | 海拔剖面+关键节点 | 户外决策核心数据 |
| P2 | 信号覆盖 | 安全相关 |
| P2 | 线路变体 | 锦上添花 |

## 输出

所有新增字段合并到 `route_full_verification_v6.json`，不要在多个文件里拆分。
